# Stack Inventory Checklist

## Detection Methods

### Page Source Analysis
- [ ] CMS platform detected
- [ ] Analytics pixels (GA4, Meta, LinkedIn)
- [ ] Chat widgets (GHL, Intercom, etc.)
- [ ] Form handlers and endpoints
- [ ] JavaScript libraries and frameworks
- [ ] Embedded booking systems

### DNS / Public Info
- [ ] Hosting provider
- [ ] CDN in use
- [ ] Email provider (MX records)
- [ ] SSL certificate info

### Behavioral
- [ ] Page load patterns (SPA vs MPA)
- [ ] Redirect flows and funnels
- [ ] Form behaviors (AJAX, page reload)
- [ ] Authentication patterns

### Career Pages
- [ ] Tools mentioned in job descriptions
- [ ] Stack listed in engineering/culture pages

### Support Channels
- [ ] Knowledge base platform
- [ ] Ticketing system
- [ ] Chat vs email vs phone

# Pain Point Analysis

| Area | Signal | Severity |
|------|--------|----------|
| Lead response | No chat widget, form-only contact | 🔴 Critical |
| Lead response | No auto-reply or acknowledgment | 🔴 Critical |
| Admin | No booking system on site | 🟡 High |
| Admin | PDF-based manual processes | 🟡 High |
| Support | FAQ is static text only | 🟡 High |
| Support | No self-service options | 🟡 High |
| Reporting | No visible analytics or dashboards | 🟢 Medium |
| Compliance | Missing required disclaimers | 🔴 Critical |
| Compliance | Privacy policy missing or outdated | 🔴 Critical |
| Content | Office hours don't match reality | 🟢 Medium |
| SEO | No schema markup on key pages | 🟢 Medium |
